US-1 — do nothing
======================

CONTEXTO
--------
Bootstrap e ambiente inicial do projeto Laravel 11 com Docker, PostgreSQL e Redis. Este é o alicerce sobre o qual todo o encurtador de URLs será construído, permitindo que o time trabalhe em um ambiente consistente e escalável.

HISTORIA
--------
Como desenvolvedor,
quero ter o projeto Laravel 11 rodando em Docker com PostgreSQL, Redis e Nginx,
para que eu possa focar em implementar features sem lidar com problemas de ambiente.

CRITERIOS DE ACEITACAO
-----------------------
1. Projeto Laravel 11 criado com Composer e estrutura padrão
2. docker-compose.yml contém serviços: app (PHP-FPM), nginx, postgres, redis
3. .env configurado com credenciais padrão (dev) apontando para containers
4. Dockerfile multistage com PHP 8.2+, extensões essenciais (pdo-pgsql, redis, curl, json, mbstring)
5. Nginx configurado para servir Laravel na porta 8080
6. OPcache habilitado em docker/php.ini
7. GET /health retorna JSON {"status":"ok","timestamp":"..."} com HTTP 200
8. GET /health retorna HTTP 503 quando PostgreSQL ou Redis estiver indisponível
9. php artisan migrate executa sem erros dentro do container
10. php artisan migrate:reset desfaz migrações corretamente
11. PHPUnit configurado e teste básico de health check passa
12. docker-compose up -d inicia todos os serviços corretamente
13. README.md documenta stack técnico e instruções de setup
14. .gitignore e .dockerignore configurados adequadamente

REGRAS TECNICAS
---------------
- Laravel 11, PHP 8.2+, PostgreSQL 15+, Redis 7+
- Imagens oficiais: php:8.2-fpm, postgres:15-alpine, redis:7-alpine, nginx:latest
- Health check valida conexão com PostgreSQL e Redis sem ORM complexo
- Logging estruturado nas tentativas de conexão
- Volumes mapeados para live reload (/app) e dados persistentes
- Docker Compose v2+
- Health checks no docker-compose para garantir readiness dos serviços

CRITERIOS DE TESTE (QA)
------------------------
API:
  - GET /health → 200 {"status":"ok"} quando todos os serviços estão disponíveis
  - GET /health → 503 {"status":"error"} quando PostgreSQL ou Redis está indisponível
  - Resposta inclui timestamp válido no formato ISO 8601

E2E:
  - docker-compose up -d inicia 4 serviços sem erro
  - docker-compose exec app php artisan migrate executa sem erro
  - curl http://localhost:8080/health retorna 200
  - docker-compose logs app mostra tentativas de conexão
  - docker-compose down limpa recursos sem erro

Unit:
  - PHPUnit executa sem erro: ./vendor/bin/phpunit
  - Teste básico passa: GET /health retorna 200
